# Test Fixture Research: WinHelp (.HLP) Files

Last Updated: 2026-04-14

Candidates for the `winhelp2rst` test suite, ranked by suitability. Criteria:
programming/technical content, clear licensing, helpdeco-compatible, direct
download availability, format coverage (WinHelp 3.1 and 4.0).

---

## 1. OpenWatcom C Library Reference (`clib.hlp`) — PRIMARY FIXTURE

- **Source**: github.com/open-watcom/open-watcom-1.9/releases (tag `w11.0c-zips`)
- **License**: Sybase Open Watcom Public License (open source, redistributable)
- **Format**: Available in both WinHelp 3.1 (Win16) and WinHelp 4.0 (Win32)
- **Downloads**:
  - `c_hlp_win.zip` (3.3 MB) — Win16 variant, WinHelp 3.1
  - `c_hlp_nt.zip` (1.9 MB) — Win32 variant, WinHelp 4.0
- **Content**: Complete C standard library function reference (printf, malloc,
  fopen, string functions, math functions, I/O, etc.). Hundreds of topics with
  cross-references, keywords, and code examples.
- **Why it's ideal**: Open source license eliminates redistribution concerns.
  Same content in both WinHelp 3.1 and 4.0 formats provides a natural A/B
  test — parse both, verify identical document model output. C library
  reference is enterprise-appropriate technical documentation. Rich enough
  to exercise all parser features (formatting, links, keywords, browse
  sequences). helpdeco handles it cleanly for ground truth.
- **Also available**: `clr.hlp` (C Language Reference), `cguide.hlp`
  (C/C++ User's Guide), `pguide.hlp` (Programmer's Guide), `wccerrs.hlp`
  (C Diagnostic Messages), `lguide.hlp` (Linker Guide), debugger and IDE
  help files. All same license, same release zips.

## 2. PuTTY 0.70 (`putty.hlp`) — MIT License, WinHelp 4.0

- **Source**: the.earth.li/~sgtatham/putty/0.70/putty.hlp
- **Companion**: the.earth.li/~sgtatham/putty/0.70/putty.cnt
- **License**: MIT — explicitly freely redistributable including documentation
- **Format**: WinHelp 4.0 (generated by Halibut, a known-correct WinHelp writer)
- **Size**: ~250 KB
- **Content**: Full PuTTY user manual — SSH client configuration, terminal
  settings, network protocols, authentication. Technical/networking content.
- **Why it's good**: MIT license is the cleanest possible. Generated by Halibut
  (not the Microsoft help compiler), so it may exercise different encoding
  choices within the WinHelp spec. Medium size, good for quick iteration.
  PuTTY dropped the .hlp file after v0.70 (2017).
- **Note**: Halibut source is also available (same author), enabling generation
  of custom test HLP files with controlled content.

## 3. Borland Turbo C++ 3.x (`TCWHELP.HLP`) — WinHelp 3.1 Stress Test

- **Source**: github.com/Balays33/Borland-Turbo-C (raw download from BIN/)
- **License**: Abandonware (Borland Turbo C++ 3.x, discontinued ~1993)
- **Format**: WinHelp 3.1
- **Size**: 4.7 MB — large file, good stress test
- **Content**: Complete Turbo C++ Windows reference — C/C++ language, standard
  library, Windows API bindings, IDE documentation.
- **Also available in same repo**:
  - `WORKHELP.HLP` (923 KB) — WorkBench help
  - `WIN31MWH.HLP` (348 KB) — Windows 3.1 multimedia API
  - `WINSIGHT.HLP` (58 KB) — small, good for smoke tests
  - `WINSPCTR.HLP` (64 KB) — WinSpector debugger
- **Why it's good**: Large WinHelp 3.1 file exercises the parser at scale.
  C/C++ compiler documentation is directly relevant content. The smaller
  companion files provide a range of complexity levels for testing.

## 4. Windows 3.1 SDK Help Files — WinHelp 3.1, API Reference

- **Source**: github.com/JackTrapper/Windows-3.1-SDK/tree/master/HELP
- **License**: Microsoft Win 3.1 SDK (1992), freely distributed as part of SDK
- **Format**: WinHelp 3.1 (1992 — the exact era the parser targets first)
- **Files**:
  - `WIN31QH.HLP` — Windows 3.1 API quick help
  - `WIN31MQH.HLP` — Multimedia API quick help
  - `PENAPIQH.HLP` — Pen API quick help
  - `MCISTRQH.HLP` — MCI string interface
- **Content**: Windows 3.1 programming API documentation. Core Win32/Win16
  function reference.
- **Why it's good**: Authentic WinHelp 3.1 format from the era when the format
  was defined. Pure programming API reference. Multiple files of varying size.
  Already on GitHub for easy access.

## 5. Win32 Programmer's Reference (`win32.hlp`) — WinHelp 4.0 Maximum Size

- **Source**: github.com/trietptm/OllyDbg-Archive/tree/master/HLP-Files
- **License**: Microsoft SDK documentation (widely archived, freeware distribution)
- **Format**: WinHelp 4.0
- **Size**: 24 MB — the largest single WinHelp file commonly available
- **Content**: The complete Win32 API reference. Thousands of topics covering
  every Windows API function, structure, message, and constant.
- **Also available in same repo**:
  - `x86eas.hlp` (475 KB) — x86 assembly reference
  - `OPCODES2.HLP` (85 KB) — x86 opcode reference (tiny, good smoke test)
  - `OLLYDBG.HLP` (296 KB) — OllyDbg debugger documentation
  - `CMDLINE.hlp` (13 KB) — minimal file, edge-case testing
- **Why it's good**: Ultimate stress test for the parser. If `win32.hlp` parses
  correctly, the parser handles anything. The smaller companion files provide
  quick-iteration test targets. The x86 opcode files are directly relevant to
  embedded/systems programming.

## Other Notable Candidates (Not Top 5)

**MASM 6.11 Help Files** (github.com/fititnt/assembly-masm):
  `ALANG.HLP` (assembly language ref), `CV.HLP` (CodeView debugger),
  `ML.HLP` (macro assembler). WinHelp 3.1 era, assembly/embedded-adjacent.

**MP8085 Microprocessor Help** (github.com/cincoutprabu/mp8085):
  `MP8085.hlp` (14 KB) — 8085 instruction set reference. Tiny, directly
  embedded-relevant. Written with Turbo C++ 3.0.

**Halibut** (chiark.greenend.org.uk/~sgtatham/halibut/):
  Open source documentation compiler that generates WinHelp .hlp files
  directly from markup. Can be used to generate custom test fixtures with
  fully controlled content for targeted regression tests.

**archive.org `sample.hlp`** (archive.org/details/microsoft_help_viewer_hlp):
  49 KB Microsoft sample HLP file. Minimal canonical example of the format.
  Good for the simplest possible integration test.
